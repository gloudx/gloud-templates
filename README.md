# Gloud — шаблоны приложений

Репозиторий-источник шаблонов для Gloud. Приложение читает `index.json`, а файлы
шаблонов тянет по raw-URL (`https://raw.githubusercontent.com/<owner>/<repo>/<branch>/<file>`).

В `.env` приложения указывается:

```
TEMPLATES_REPO=<owner>/<repo>@<branch>
```

## Структура

```
index.json              — каталог шаблонов
templates/<id>.json     — файлы шаблонов
```

### index.json

```json
{
  "version": 1,
  "templates": [
    {
      "id": "demo",
      "title": "Название",
      "description": "Короткое описание",
      "icon": "bi-box",
      "version": 1,
      "file": "templates/demo.json",
      "hasSampleData": false
    }
  ]
}
```

- `version` шаблона — увеличивай при изменении содержимого; приложение скачает
  новую версию при синхронизации.
- `hasSampleData: true` — файл содержит и тестовые данные (`schema_records`,
  `schema_values`); при импорте пользователь сможет включить их галочкой.

### Файл шаблона (`templates/<id>.json`)

Это бандл формата Gloud:

```json
{
  "kind": "template",
  "version": 1,
  "tables": { "schema_types": [...], "schema_fields": [...], ... }
}
```

Получить такой файл проще всего через приложение: **Настройки → Система →
Експорт шаблону** (секреты — telegram-токены, webhook-ключи, SMTP — вырезаются
автоматически). Чтобы добавить тестовые данные, экспортируй с данными и проставь
`hasSampleData: true` в `index.json`.

## Как добавить шаблон (новый)

1. Положи файл в `templates/<id>.json`.
2. Добавь запись в `index.json` с `version: 1`.
3. Закоммить и запушь — приложения подтянут при следующей синхронизации
   (фоново при старте или по кнопке «Обновить»).

## Как выпустить ОБНОВЛЕНИЕ шаблона

Обновление применяется к уже работающей установке **без потери данных** —
меняется только конфиг (сущности, поля, скрипты, подання, дашборди, пивоти,
тригери, розклади). Данные пользователя сохраняются.

### Золотое правило (иначе сломаются данные пользователей)

Обновление сопоставляет строки конфига **по их `id`**, чтобы записи пользователя
остались привязанными к правильным сущностям/полям. Поэтому:

> Развивай шаблон в **одной и той же** установке-«мастере» и переэкспортируй её.
> **Нельзя** пересоздавать/обнулять мастер-установку перед экспортом — это
> перенумерует `id`, и у пользователей обновление превратит «изменение» в
> «удалить + создать», уничтожив данные.

### Шаги

1. **Мастер-установка.** Держи отдельную установку, развёрнутую из этого шаблона
   (через библиотеку). В ней вноси изменения: добавляй/меняй/удаляй сущности,
   поля, скрипты, подання, дашборди и т.д.
2. **Экспорт.** Настройки → Система → **Експорт шаблону** → скачается
   `gloud-template.json` (только конфиг; секреты вырезаны).
3. **Замени** содержимое `templates/<id>.json` этим файлом.
4. **Подними `version`** этого шаблона в `index.json` (напр. `3` → `4`). Это и
   есть триггер обновления (установка видит апдейт, когда версия в репо больше
   установленной).
5. Закоммить и запушь:
   ```bash
   git add index.json templates/<id>.json
   git commit -m "tpl <id>: v4 — добавлено поле X, удалено Y"
   git push
   ```
6. На установке: Настройки → Система → **«Обновить»** → на карточке шаблона
   появится бейдж **«Оновлення v3→v4»** → **Оновити** → превью покажет
   добавлено/изменено/удалено (с предупреждением о потере данных, если поле или
   сущность удаляются) → подтвердить.

### Что нужно помнить

- **Версия** — целое число, должна только **расти**.
- **Удаление в шаблоне = удаление в установке.** Удаление поля теряет его
  значения, удаление сущности — все её записи. Это показывается в превью и
  требует явного подтверждения галочкой.
- **Не трогается при обновлении:** `app_settings` (название, иконка, цвет,
  главная страница — остаются у каждой установки) и данные пользователей.
- **Секреты:** локальные telegram-токени/webhook-ключі установки сохраняются;
  новые вебхуки получают свежие ключи.
- **In-place обновление доступно** только если установка была развёрнута из
  этого шаблона через библиотеку (стоит маркер происхождения). Если конфиг
  правили вручную или установлен другой шаблон — обновление не предложится,
  нужен полный реимпорт.

> Не клади сюда секреты: репозиторий публичный.
